Definición de rutas

Como se puede ver primero esta la ruta para agregar información por medio del metodo POST. Despues los datos seran recopilados por medio de un formulario y posteriormente seran enviados a la base de datos, por ultimo estos datos seran retornados al servidor. Los datos recopilados son name, price, quantity y brand.

Como se muestra en la imagen primero se define una ruta que recibe una variable de tipo string por medio de PUT, despues se obtiene el producto solicitado, se verifica si la lista no esta vacia y realiza la respectiva modificación.

Se define ruta para recibir una varible de tipo string por medio de DELETE posteriormente se obtiene el producto, se verifica si la lista no esta vacia y se realiza la accion requerida.

A continuacion se muestra una aplicación con la cual se pueden ejecutar todos los metodos de HTTP pero en nuestro caso solo se mostraran ejemplos con el metodo PUT Y DELETE. Se hara uso de esta aplicación porque HTML no te permite hacer peticiones mediante PUT Y DELETE.
